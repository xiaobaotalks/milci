核心特性

11 大功能模块，为开发者量身打造

🧠

四层记忆系统

checkpoint（会话）/ MEMORY（项目）/ notes（草稿）/ history（日志），启动自动恢复，跨会话持久化。

📝

任务级 Checkpoint

每次工具调用自动写入任务 checkpoint，记录目标 / 步骤 / 已修改文件 / 阻塞问题，崩溃可续。

🔀

多会话管理

每个会话独立的 history / checkpoint / 压缩状态，通过 /session 命令切换，隔离不同项目的上下文。

🔄

多 Provider 故障转移

配置多组 API Key，主 Provider 失败（401/429/5xx）时自动切换，保证对话不中断。

📦

分层摘要压缩

三档触发（60% / 80% / 95%），滚动窗口自动维护最近 N 轮，摘要层持久化，重启不丢。

🤖

13 个模型自动识别

内置 MiMo 1M / GPT-4o / Claude / DeepSeek / GLM-4 / Moonshot 等上下文窗口映射，启动自动设置。

🛠️

工具调用 + MCP

内置 readFile / writeFile / runShell / git，支持 MCP 风格外部工具声明，管道安全拦截。

📚

技能库自动蒸馏

/distill 从历史对话挖掘高频工作流，top-2 匹配技能自动注入 System Prompt，提升回答质量。

🔍

项目索引与问答

/index 扫描 ts/js/json/md 提取函数/导入/导出；/ask 基于索引进行代码问答，支持 TF-IDF 检索。

🌐

MCP Server 模式

通过 --mcp 参数启动为 MCP Server，可被 OpenClaw、Cursor、Claude 等宿主调用，暴露完整 Agent 能力。

🎨

彩色终端 UI

chalk 驱动的彩色渲染，工具调用结果带边框和状态图标，助手回复带标题，一眼识别信息类型。

🔒

安全加固

白名单命令拦截，路径安全校验，操作审计日志，API Key 可加密存储（keytar），保障敏感操作。

快速开始

3 步即可在任意终端运行

1克隆并安装依赖

git clone https://github.com/xiaobaotalks/mi-cc.git
cd mi-cc
npm install

2配置 API（默认对接小米 MiMo）

cp .env.example .env
# 编辑 .env，填入 API_KEY，MODEL 默认 mimo-v2.5-pro

3启动

npm run dev            # 开发模式
# 或
npm link && mi-cc      # 全局安装后直接使用
mi-cc --mcp            # 以 MCP Server 模式运行
