mi-cc

自带四层记忆、多模型故障转移、自动压缩的 AI 编程助手。
本地终端体验 + MCP Server 模式，让 OpenClaw/Cursor/Claude 获得专属编程能力。

核心特性

8 大核心能力，为开发者量身打造

🧠

四层记忆系统

checkpoint（会话）/ MEMORY（项目）/ notes（草稿）/ history（日志），启动自动恢复，跨会话持久化。

💾

任务级 Checkpoint

每次工具调用自动写入任务 checkpoint，记录目标 / 步骤 / 已修改文件 / 阻塞问题，崩溃可续。

🔄

多 Provider 故障转移

配置多组 API Key，主 Provider 失败（401/429/5xx）时自动切换，保证对话不中断。

📦

分层摘要压缩

三档触发（60% / 80% / 95%），滚动窗口自动维护最近 N 轮，摘要层持久化，重启不丢。

🎯

完整任务执行

agent_execute 自主规划 + 多轮工具调用完成完整编程任务，而非仅支持原子操作。

📚

技能库自动蒸馏

/distill 从历史对话挖掘高频工作流，top-2 匹配技能自动注入 System Prompt，提升回答质量。

🔍

项目索引与问答

/index 扫描代码提取函数/导入/导出；/ask 基于索引进行代码问答，支持 TF-IDF 检索。

🌐

MCP Server 模式

通过 --mcp 参数启动，暴露完整 Agent 能力给 OpenClaw、Cursor、Claude，开箱即用。

为什么选择 mi-cc？

与普通 MCP 工具的核心差异

对比维度 mi-cc 普通 MCP 工具
🧠 记忆系统 四层记忆（checkpoint/MEMORY/notes/history） 无
🔄 模型切换 多 Provider 故障转移，401/429/5xx 自动切换 依赖宿主模型，无切换能力
📦 上下文压缩 分层摘要 + 滚动窗口，超长对话自动压缩 依赖宿主处理
🎯 任务执行 agent_execute 自主规划 + 多轮工具调用 仅支持原子操作（读/写/执行）
📚 技能库 /distill 经验蒸馏，自动挖掘工作流 无
🔍 项目索引 /index 扫描代码，/ask 语义问答 无
💾 状态持久化 任务级 checkpoint，崩溃可续 无
🚀

本地编程助手

无缝终端体验，超长对话自动压缩，任务断点续跑，不依赖任何宿主

🔗

MCP 服务端

让 OpenClaw/Cursor/Claude 获得专属编程能力，自主规划完成完整任务

⚡

多模型保障

配置多组 API Key，自动故障转移，单个模型故障不中断任务执行

快速开始

3 步即可在任意终端运行

1克隆并安装依赖

git clone https://github.com/xiaobaotalks/mi-cc.git
cd mi-cc
npm install

2配置 API（默认对接小米 MiMo）

cp .env.example .env
# 编辑 .env，填入 API_KEY，MODEL 默认 mimo-v2.5-pro

3启动

npm run dev            # 开发模式
# 或
npm link && mi-cc      # 全局安装后直接使用
mi-cc --mcp            # 以 MCP Server 模式运行
