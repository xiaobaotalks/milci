核心特性

8 大核心能力，为开发者量身打造

🧠

四层记忆系统

checkpoint（会话）/ MEMORY（项目）/ notes（草稿）/ history（日志），启动自动恢复，跨会话持久化。

💾

任务级 Checkpoint

每次工具调用自动写入任务 checkpoint，记录目标 / 步骤 / 已修改文件 / 阻塞问题，崩溃可续。

🔄

多 Provider 故障转移

配置多组 API Key，主 Provider 失败（401/429/5xx）时自动切换，保证对话不中断。

📦

分层摘要压缩

三档触发（60% / 80% / 95%），滚动窗口自动维护最近 N 轮，摘要层持久化，重启不丢。

🎯

完整任务执行

agent_execute 自主规划 + 多轮工具调用完成完整编程任务，而非仅支持原子操作。

📚

技能库自动蒸馏

/distill 从历史对话挖掘高频工作流，top-2 匹配技能自动注入 System Prompt，提升回答质量。

🔍

项目索引与问答

/index 扫描代码提取函数/导入/导出；/ask 基于索引进行代码问答，支持 TF-IDF 检索。

🌐

MCP Server — 接入 OpenClaw / Cursor / Claude

通过 --mcp 启动，暴露完整 Agent 能力给宿主 AI 助手，开箱即用。

为什么接入 mi-cc？

✓ 多模型故障转移：401/429/5xx 自动切换，不中断任务
✓ 四层记忆持久化：宿主的对话自动写入 checkpoint/MEMORY
✓ 分层摘要压缩：超长对话自动压缩，节省 Token
✓ 完整任务执行：agent_execute 自主规划 + 多轮工具调用
✓ 项目索引问答：接入后自动建立索引，支持 /ask 语义问答

OpenClaw 配置

{
  "mcpServers": {
    "mi-cc": {
      "command": "mi-cc",
      "args": ["--mcp"],
      "env": { "API_KEY": "your_key" }
    }
  }
}

Skill 模式 — 接入 Hermes / Agent 框架

mi-cc 可作为 Skill 被 Agent 框架调用，提供专业领域的编程能力。

Skill 的核心价值

✓ 技能蒸馏：/distill 从历史对话自动挖掘工作流
✓ 智能匹配：输入时 top-2 相关技能自动注入 System Prompt
✓ 可扩展：skill-lib.md 声明式添加新技能
✓ 跨会话：技能库跨会话共享，持续积累

Skill 匹配模式

# 启动 Skill 匹配模式
mi-cc --skill "重构 TypeScript 项目"

# Agent 调用示例
@mi-cc 重构这个项目的错误处理逻辑

技能库示例（skill-lib.md）

## 重构 TypeScript
当用户说"重构"、"优化代码"时触发
- 步骤：分析依赖 → 提取类型 → 移除冗余 → 添加泛型
- 工具：readFile, writeFile, runShell(npx tsc)
使用场景： Hermes 中作为编程专家，处理复杂重构任务

快速开始

3 步即可在任意终端运行

1克隆并安装依赖

git clone https://github.com/xiaobaotalks/mi-cc.git
cd mi-cc
npm install

2配置 API（默认对接小米 MiMo）

cp .env.example .env
# 编辑 .env，填入 API_KEY，MODEL 默认 mimo-v2.5-pro

3启动

npm run dev            # 开发模式
# 或
npm link && mi-cc      # 全局安装后直接使用
mi-cc --mcp            # 以 MCP Server 模式运行
